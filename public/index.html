General Help

Portal Security

Plembfin is a self-hosted watch-state bridge. Access to this dashboard is gated by a local username and password session.

Defaults: admin / admin on first run. Override by setting ADMIN_USERNAME and ADMIN_PASSWORD environment variables.

Every /api/* request is verified server-side — unauthenticated requests are rejected. External integrations (webhooks) authenticate with the API key shown after sign-in.

Plex Setup

Server URL, token, and username

Plex Integration Guide

Server & Token

Plex URL: Typically http://127.0.0.1:32400 if local. Otherwise, use your server's IP address with port 32400.

Plex Token: Sign in to Plex Web, open Get Info on any library item, and click View XML. Copy the alphanumeric token value following X-Plex-Token= at the end of the URL.

Unwatched/Unscrobble Sync: Plex does not support unscrobbling via webhooks. To sync unwatched status instantly, run our local daemon script: node scripts/plexWebSocketMonitor.js.

Emby Setup

API key, URL, and user ID

Emby Integration Guide

API Key & User ID

Emby URL: Typically http://localhost:8096 or your secure reverse proxy domain (e.g. https://emby.example.com).

API Key: Generate a key in Emby Settings ➔ Advanced ➔ API Keys.

User ID: Go to Emby settings ➔ Users, click your profile, and copy the string value following ?userId= in the browser URL.

Webhook Setup: Add our webhook endpoint in Emby webhooks settings, checking Playback (Start, Pause, Unpause, Stop), Users (Mark Played, Mark Unplayed), and Send All Properties.

Jellyfin Setup

API key, URL, and user ID

Jellyfin Integration Guide

API Key & User ID

Jellyfin URL: Typically http://localhost:8096 or your secure reverse proxy domain.

API Key: In admin dashboard, go to Advanced ➔ API Keys, generate a new token named Plembfin Bridge.

User ID: Go to Administration ➔ Users, click your username, and copy the alphanumeric ID following /users?userId= in the address bar.

Webhook Setup: Install the official Webhook plugin in Jellyfin. Create a Generic Webhook targeting /api/webhook, checking Playback Start/Progress/Stop and User Data Saved events. Check Send All Properties.

Backup Guide

Disaster Recovery

Local & Synced Folders

Backups are saved to data/backups/watch-history. You can point the folder destination to a OneDrive or Dropbox sync folder to automate off-host backups.

Backblaze B2 Mirror

Create a private B2 bucket on backblaze.com, locate its region code (e.g. us-west-004), generate an application key, and enter the credentials.

OneDrive Mirror

Register an app in Entra portal (Accounts in any organizational directory and personal MS accounts), enable public client flows, and enter the Azure app client ID.

Dropbox Mirror

Create an app in Dropbox App Console with files.content.write/read permissions and enter the App Key and App Secret.

Restore Guide

                        How to restore
                      


                      
What a restore does

                      

                        The selected backup becomes the single source of truth. Plembfin runs an authoritative restore:
                      

                      
    
                        
  1. Wipes the local watch history, playstate and resume progress, and reloads them from the backup.
    2. 
                        
  2. Pushes that state out to every connected app (Plex, Emby, Jellyfin).
    3. 
                        
  3. Stamps a restore watermark so the background sync can never re-import the apps' state afterwards.
    4. 
                      

                      

                        While the restore is running, the cron sync and incoming webhooks are ignored, so nothing is written to watch history mid-restore. Watch dates are always preserved — an item already watched on an earlier date is never re-added as watched today.
                      


                      
Reconcile tracked items

                      

                        Pushes only the items the backup knows about. Fast. Any extra items the apps have marked watched but the backup never tracked are left untouched. Choose this for a normal point-in-time restore.
                      


                      
Full wipe then push

                      

                        First marks every currently-watched item on each app as unwatched, then re-applies only the backup's watched set, so the apps end up matching the backup exactly. Slower (it touches every watched item on every app). Choose this when the apps contain watched state you want removed.
                      


                      
Restore onto New Instance

                      
    
                        
  1. Install a fresh Plembfin.
    2. 
                        
  2. Go to Backups → Settings and configure remote destinations.
    3. 
                        
  3. Go to Backups → Restore and click "Watch History Wipe / Restore" on a backup from your remote destination.
    4.

Synchronization Help

                        Queues & Jobs
                      

                      

                        Background Worker: Runs every 60 seconds to poll active playback sessions, commit watches above 90% progress, dispatch pending sync jobs, and poll Plex for unwatched removals.
                      

                      

                        Resume Progress Sync: Playback stopped below 90% and above 1 minute stores the current resume offset tick count and replicates it to all other platforms.
                      

                      

                        Loop Prevention: An in-memory loopStore tracks dispatched watch states to prevent feedback echo loops when media server webhooks trigger.
