Local Restore Guide

                        Disaster Recovery
                      


                      
Two backup types

                      

                        Watch-history backups contain watched state, playstate, and resume progress. Use them to roll watch history back to a known point and push that state to Plex, Emby, and Jellyfin.
                      

                      

                        Plembfin backups contain the full Plembfin database data and saved settings. They are encrypted because they can include API keys and server credentials.
                      


                      
Watch-history restore

                      

                        The selected watch-history backup becomes the source of truth:
                      

                      
    
                        
  1. Wipes the local watch history, playstate and resume progress, and reloads them from the backup.
    2. 
                        
  2. Pushes that state out to every connected app (Plex, Emby, Jellyfin).
    3. 
                        
  3. Stamps a restore watermark so background sync does not re-import the previous app state.
    4. 
                      

                      

                        While the restore is running, the cron sync and incoming webhooks are ignored, so nothing is written to watch history mid-restore. Watch dates are always preserved — an item already watched on an earlier date is never re-added as watched today.
                      


                      
Reconcile tracked items

                      

                        Pushes only the items the backup knows about. Fast. Any extra items the apps have marked watched but the backup never tracked are left untouched. Choose this for a normal point-in-time restore.
                      


                      
Full wipe then push

                      

                        First marks every currently-watched item on each app as unwatched, then re-applies only the backup's watched set, so the apps end up matching the backup exactly. Slower (it touches every watched item on every app). Choose this when the apps contain watched state you want removed.

Sync Issues Help

                        Understanding Issues
                      


                      

                        What are Sync Issues?
                        

                          This section shows any problems with propagating watched states between your media servers. Click on any issue to view detailed telemetry about what went wrong.
                        

                      


                      

                        Issue Categories
                        

                          Missing Dispatch Telemetry: No logging data recorded. These likely synced successfully but the logs are missing. Safe to clear.
                        

                        

                          Plex Match Issues: Plex couldn't find the media item. Check Plex metadata and external IDs.
                        

                        

                          Emby/Jellyfin Match Issues: Plex found it but Emby/Jellyfin couldn't. Check their metadata and IDs.
                        

                        

                          Unresolved Sync Issues: Other sync problems detected. Click to view telemetry for error details.
                        

                      


                      

                        How to Fix
                        

                          Click "Retry" on an issue to re-attempt the sync and see detailed error messages. Click "Fix" to manually adjust the match if the item wasn't found.
                        

                      

                    

                  

                


                
                

                  

                    

                      
                      

                        

                      

                    

                  


                  

                    

                      

                        
Sync History Help

                        Recent Propagations
                      


                      

                        What is Sync History?
                        

                          Recent watched-state propagation attempts. Shows source, action, timestamp, and target sync results for every automatic and manual sync.
                        

                      


                      

                        How Sync Works
                        

                          When you mark something as watched (on any platform), Plembfin automatically propagates that state to your other configured servers every 60 seconds.
                        

                        

                          Resume positions sync similarly. Use "Force Sync" to immediately push any pending updates without waiting.

Sync Tools Help

                        Manual Control
                      


                      

                        Run Cron Sync: Executes the scheduled sync job immediately, checking for outstanding propagations and updating resume positions across all platforms.
                      

                      

                        Force Sync: Forcefully propagates all pending watched-state changes to your other platforms right away, bypassing the 60-second scheduled interval. Use this to immediately sync a watch or unwatched state.
                      

                      

                        Stop Sync: Halts a currently running force sync operation. Appears only when a sync is actively running.
                      

                      

                        Refresh: Reloads the sync issues list to check for any new or resolved issues without waiting for the next scheduled check.
                      

                    

                  

                

              


              

                

                  

                    

                      

                        

                          
Server Logs

                          Terminal output from the local server, plus frontend diagnostics
                        

                        

                          
                          
                        

                      

                      

                        

                          

                            
                            
                            
                          

                          Server Console
                          

                        

                        

                      

                    

                  


                  

                    

                      

                        
Understanding Logs

                        What you might see
                      

                      
Logs are captured in memory from the moment the server starts. They reset on restart and hold up to 1,000 entries. The panel refreshes every 3 seconds automatically.


                      

                        Normal activity
                        
    
                          
  • Scheduled Sync: starting background sync workflow — the per-minute background worker has fired. Completely normal; you will see this every 60 seconds.
    • 
                          
  • Plex / Emby / Jellyfin: checking recently watched — each server's recent history is being polled for new watches.
    • 
                          
  • detected new watched item — an item was found on a server that isn't yet in Plembfin's history. It will be imported and synced.
    • 
                          
  • Resume Sync: … skipped (item is watched / unchanged) — a partial-play resume position was checked but not propagated because the item is already fully watched or the position hasn't changed. Normal.
    • 
                          
  • Webhook received — a play event arrived from Plex, Emby, or Jellyfin in real time.
    • 
                        

                      


                      

                        Pre-restore skips
                        
skipped pre-restore item — After importing a backup, Plembfin sets a restore timestamp. Any item whose play date falls at or before that timestamp is skipped during scheduled sync to prevent the same history being imported twice. These entries are expected after a restore and will stop appearing once the timestamp clears or the items age out.

                      


                      

                        Warnings and errors
                        
    
                          
  • [WARN] — A non-critical issue, such as a missing metadata field or a server returning unexpected data. Sync continues.
    • 
                          
  • [ERROR] — Something failed, such as a network error reaching Plex, Emby, or Jellyfin, or a failed sync dispatch. Check your server URLs and API keys in Settings → Apps if these repeat.
    • 
                          
  • Plex / Emby / Jellyfin ERROR: failed to fetch watched items — The scheduled sync could not reach that server. Usually a network or credential issue.
    • 
                          
  • loop detected — A sync propagation triggered a webhook back from the target server. The loop guard blocked it. Normal when all three platforms are connected.
    • 
                        

                      


                      

                        Force sync logs
                        
When you run a force sync from Settings → Tools, progress is streamed here in real time: how many items were fetched from each server, how many were reconciled, and the result of each mark-played or mark-unplayed call. A Force Sync: process complete line confirms it finished.

                      


                      
Use Clear Logs to wipe the in-memory buffer, or Copy Logs to paste them when reporting an issue.
