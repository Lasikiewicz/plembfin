Integrity Help

Database Care
Run System Diagnostic

Runs nine live checks in sequence and reports each as Online, Warnings Detected, Failed, or Not Configured, with a "Fix" tip and a link to the relevant settings page for anything that isn't healthy:

  • Watch History API — confirms the local SQLite database responds.
  • Server Configuration — confirms saved media-server/config settings can be read back.
  • Webhook Listener Endpoint — pings /api/webhook and shows the last event received, if any.
  • Scheduled Cron Job — checks the per-minute background worker fired recently.
  • Outbound Playstate Sync — scans the 5 most recent history rows' dispatch telemetry for failures.
  • Plex / Emby / Jellyfin Media Server — live connection test against each configured server.
  • Plex Realtime Notifications — confirms the WebSocket notification listener can connect (unwatch sync falls back to the periodic poll if this fails).
  • Cross-Platform Library Matching — summarizes current "no matching item found" sync failures per platform.
Refresh Sync Health

Loads a lighter-weight snapshot: watch history/playstate/playback-progress row counts, database size on disk, and outbound request pressure (requests, throttles, and cooldowns) per media-server host — useful for spotting rate-limiting or an oversized database before running a large Force Sync.

Database Repairs Help

Database Care
Repair History Rows

Scans your 300 most recent history rows for ones missing a media type or an optimized (locally cached) poster. Rows with no type are guessed as an episode or movie from the title pattern (e.g. S01E01), then a poster is resolved from the cache, the configured media server, or TMDB and saved to local storage. Runs in passes of up to 10 rows and stops automatically once a pass makes no changes.

Clean Duplicate History Rows

Groups all watch-history rows by media (title/type/season/episode). Where a group has more than one row, it keeps only the most recently watched row and deletes the rest. Use this if the same play was recorded twice (e.g. by both a webhook and the scheduled poll).

Backup Guide

Disaster Recovery
Local Watch History Backups

Unencrypted daily snapshots of watched state, playstate, and resume progress, saved to data/backups/watch-history on the server running Plembfin. No passphrase needed — "Save Schedule" and "Back Up Now" work as soon as backups are enabled.

Local Plembfin Backups

A full encrypted export of the database and saved settings (including server credentials and API keys), encrypted in your browser with the passphrase you enter — Plembfin never sees or stores the plaintext backup. You need at least 12 characters. Losing the passphrase means the backup cannot be decrypted; there is no recovery.

Remember passphrase

Scheduled Plembfin backups run on the server with no browser open, so they need the passphrase available server-side. Checking "Remember passphrase" stores it in data/config.json so the schedule can run unattended; leaving it unchecked keeps the passphrase browser-only, but then only "Back Up Now" works (the schedule stays disabled) since there's no passphrase for the server to encrypt with.

Remote Destinations

Configure a Backblaze B2 bucket (region/endpoint, bucket name, and an application key scoped to it) as an upload target. Every enabled destination receives a copy of each remote backup run.

Remote Watch History / Plembfin Backups

Mirror the same two backup types to every enabled remote destination on their own schedule and retention. Remote Plembfin backups use the same passphrase and "remember" behavior as the local encrypted backup above.

Local Restore Guide

                        Disaster Recovery
                      


                      
Two backup types

                      

                        Watch-history backups contain watched state, playstate, and resume progress. Use them to roll watch history back to a known point and push that state to Plex, Emby, and Jellyfin.
                      

                      

                        Plembfin backups contain the full Plembfin database data and saved settings. They are encrypted because they can include API keys and server credentials.
                      


                      
Watch-history restore

                      

                        The selected watch-history backup becomes the source of truth:
                      

                      
    
                        
  1. Wipes the local watch history, playstate and resume progress, and reloads them from the backup.
    2. 
                        
  2. Pushes that state out to every connected app (Plex, Emby, Jellyfin).
    3. 
                        
  3. Stamps a restore watermark so background sync does not re-import the previous app state.
    4. 
                      

                      

                        While the restore is running, the cron sync and incoming webhooks are ignored, so nothing is written to watch history mid-restore. Watch dates are always preserved — an item already watched on an earlier date is never re-added as watched today.
                      


                      
Reconcile tracked items

                      

                        Pushes only the items the backup knows about. Fast. Any extra items the apps have marked watched but the backup never tracked are left untouched. Choose this for a normal point-in-time restore.
                      


                      
Full wipe then push

                      

                        First marks every currently-watched item on each app as unwatched, then re-applies only the backup's watched set, so the apps end up matching the backup exactly. Slower (it touches every watched item on every app). Choose this when the apps contain watched state you want removed.

Sync Tuning Help

                        Understanding the Defaults
                      


                      

                        What This Controls
                        

                          These four values drive the heuristics behind watched-state detection and outbound sync. Every field is optional — leaving a field blank keeps the built-in default (or a matching environment variable, if one is set on the server).
                        

                      


                      

                        Fields
                        

                          Watched Threshold: playback progress percentage at which a play counts as watched. Default 90%.
                        

                        

                          Minimum Resume Position: playback position, in seconds, below which a stopped play is too short to save as a resume point. Default 60s.
                        

                        

                          Active Session TTL: how long a "now playing" session is kept without an update before it's considered stale. Default 5 minutes.
                        

                        

                          Outbound Request Timeout: how long Plembfin waits for a response from Plex, Emby, or Jellyfin before giving up. Default 10 seconds.

Sync Tools Guide

                        Detailed Tool Information
                      


                      

                        Repair Recent Items
                        
Performs a rapid incremental scan of recent watch activity and active sessions across Plex, Emby, and Jellyfin.
When to use: If a recent watch wasn't automatically picked up, run this for a quick fix without waiting or scanning full libraries.

                      


                      

                        Full Sync Watchstates
                        
Scans the complete recorded Plembfin watch archive and pushes all historical watched events back into configured Plex, Emby, and Jellyfin watchlists.
When to use: Essential after connecting a new server, rebuilding a library, or restoring watch history from a backup.

                      


                      

                        Force Full Sync
                        
Immediately triggers an end-to-end watch state reconciliation across all connected media servers, evaluating watch timestamps and play states.
When to use: Use "Preview Force Sync" to simulate and inspect planned actions before writing.

                      


                      

                        Stop / Reset Sync
                        
Cancels an in-progress Force Full Sync background job or clears an orphaned sync lock file if the server restarted mid-sync.

Sync Issues Help

                        Understanding Issues
                      


                      

                        What are Sync Issues?
                        

                          This section shows any problems with propagating watched states between your media servers. Click on any issue to view detailed telemetry about what went wrong.
                        

                      


                      

                        Issue Categories
                        

                          Missing Dispatch Telemetry: No logging data recorded. These likely synced successfully but the logs are missing. Safe to clear.
                        

                        

                          Plex Match Issues: Plex couldn't find the media item. Check Plex metadata and external IDs.
                        

                        

                          Emby/Jellyfin Match Issues: Plex found it but Emby/Jellyfin couldn't. Check their metadata and IDs.
                        

                        

                          Unresolved Sync Issues: Other sync problems detected. Click to view telemetry for error details.
                        

                      


                      


                      

                        How to Fix
                        

                          Click "Retry" on an issue to re-attempt the sync and see detailed error messages. Click "Fix" to manually adjust the match if the item wasn't found.
                        

                      


                      

                        Refresh Issues
                        

                          Click "Refresh" in the Sync Issues header to immediately reload unresolved sync issues and match reports.
                        

                      

                    

                  

                


                
                

                  

                    

                      
                      

                        

                      

                    

                  


                  

                    

                      

                        
Sync History Help

                        Recent Propagations
                      


                      

                        What is Sync History?
                        

                          Recent watched-state propagation attempts. Shows source, action, timestamp, and target sync results for every automatic and manual sync.
                        

                      


                      

                        How Sync Works
                        

                          When you mark something as watched (on any platform), Plembfin automatically propagates that state to your other configured servers every 60 seconds.
                        

                        

                          Resume positions sync similarly. Use "Force Full Sync" to immediately push any pending updates without waiting.

Logs Guide

                        System Diagnostics & Telemetry
                      


                      

                        

                          All Logs
                          
Unified stream combining both server-side background process activity and browser-side UI telemetry in chronological order.

                        


                        

                          Plex WebSockets
                          
Captures real-time WebSocket events pushed directly from Plex Media Server, including live playback session updates, pause/resume events, and unwatched media detection.

                        


                        

                          Outbound Sync
                          
Logs outbound playstate dispatches to Plex, Emby, and Jellyfin. Records mark-played and mark-unplayed calls, resume position syncs, loop-prevention checks, and target API responses.

                        


                        

                          Scheduled Polls
                          
Tracks the per-minute background worker ticks that poll connected servers for recent watch history, evaluate 90% watched thresholds, and process periodic sync queues.

                        


                        

                          System Logs
                          
Records core application lifecycle events, database initialization, configuration changes, scheduled task execution, and system errors or warnings.

                        


                        

                          Downloading & Exporting
                          
Use Download Logs to save a full .log file containing complete backend and frontend diagnostic history for bug reporting and troubleshooting.
